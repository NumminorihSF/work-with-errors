How to live with errors in JavaScript

by Konstantin Petryaev

What is error

  • Error
  • throw
  • EventEmitter#emit('error')
  • process.on('error')
  • window.onerror
  • window.addEventListener('error')
  • try, catch, finally

We don't love them. Really. Because we see it in runtime (usually).

Prevent runtime error

  • Docs
  • "use strict"
  • ESLint
  • Flow / TypeScript
  • Code Review
  • Code Style
  • Tests*, Errors*

* in production runtime.

Throw

  1. throw null;
  2. throw 11;
  3. throw 'We have some problem';
  4. throw new Error('We have some problem');

Use only 4th.

One of the reasons is that usually error handlers waiting for Error's instance.

Try/Catch/Finally

			try {
			  throw new Error('My error');
			} catch (error) {
			  console.error(error);
			}
			// My error

Try/Catch/Finally

			try {
			  return 1;
			} catch (error) {
			  return 2;
			} finally {
			  return 3;
			}
			// 3

Try/Catch/Finally

			try {
			  throw ...;
			} catch (error) {
			  if (error instanceof MyError) console.error('My error');
			  else if (error.name === 'Error') console.error('Created error');
			  else console.error('Engine error');
			}

Promise

			new Promise(() => {
			  throw new Error();
			})
			  .catch(err => console.error(err) })
			  .then(() => console.log('all if ok'));
